# Karatag (カラタグ)

歌声と歌詞テキストから、1/100秒単位の精密なタイムスタンプ（カラオケ字幕形式）を自動生成するAIツールです。

## 特徴
- **ハイブリッド解析:** AI（Whisper）が行の開始位置を見つけ、別のAI（Wav2Vec2）が1文字ずつのタイミングを確定させます。
- **物理分割アライメント (New):** 間奏タグを境界として楽曲を分割解析。長い間奏や繰り返しの多い曲での「ワープ」を物理的に封じ込めます。
- **インテリジェント自己診断 (New):** 正解データがなくても統計的な不自然さを検知。ログの末尾に改善策（タグ挿入や設定変更）を具体的に提示します。
- **詳細実況ログ (New):** AIが音声をどう聴き取ったか、なぜそのアンカーを却下したか等、全内部プロセスを可視化します。
- **読み指定対応:** `[漢字|かな]` 形式で、難読漢字や英語の読みを自由に指定できます。
- **冒頭スキップ:** `--ignore` オプションで、曲の前の長いセリフや無音をカットして解析できます。

## 1. セットアップ（初心者向け）

### 必要条件
- **Python 3.9 以上** がインストールされていること。
- **ffmpeg** がインストールされていること。（音声の読み込みに必要です）

### 環境構築の手順
ターミナル（またはコマンドプロンプト）を開き、プロジェクトのフォルダで以下を入力してください。

```bash
# 1. 仮想環境（ツール専用の部屋）を作る
python3 -m venv .venv

# 2. 仮想環境に入る
# macOS / Linux の場合:
source .venv/bin/activate
# Windows の場合:
.venv\Scripts\activate

# 3. 必要なライブラリをインストールする
pip install -r requirements.txt

### Windows環境での注意点 (ctc-segmentation のビルド失敗)
Windowsでは、`ctc-segmentation` のインストール中に `Microsoft Visual C++ 14.0 or greater is required` というエラーが発生する場合があります。このツールはC++のビルド環境を必要とするため、以下の手順で対応してください。

1. [Microsoft C++ Build Tools](https://visualstudio.microsoft.com/visual-cpp-build-tools/) をダウンロードして実行します。
2. インストーラーのワークロード一覧から **「C++ によるデスクトップ開発」** (Desktop development with C++) にチェックを入れてインストールします。
3. インストール完了後、**ターミナルを再起動**してから再度 `pip install -r requirements.txt` を実行してください。
```

## 2. 使い方

必ず上記の **仮想環境に入った状態** で実行してください。

### 基本的な実行
```bash
python -m src.cli "音声ファイル.wav" "歌詞ファイル.txt"
```
出力結果は `音声ファイル.result.txt` という名前で保存されます。
※解析中の詳細なプロセスは常に `karatag_debug.log` に記録されます。

### 便利なオプション
- `--ignore 秒数`: 楽曲冒頭のセリフなどを無視します。
- `--preset プリセット名`: 曲のタイプを指定して一括設定します（`standard`, `fast`, `ballad`）。
- `--eval 正解.kra`: 既存のデータと精度を比較して詳細な MAE レポートを表示します。
- `--export-midi`: 歌詞のタイミングとピッチを統合した MIDI ファイルを出力します（自動採譜）。
- `-o 出力名.txt`: 保存するファイル名を指定します。

## 3. 歌詞ファイルの書き方（重要）

Karatag は特殊な記法を使って AI にヒントを与えることができます。

### ① 読み方の指定 `[表示|かな]`
AIが読み間違えそうな漢字や、特殊な歌い方に使います。
- `[本当|ほんとう]` のように指定すると、読みの文字数に合わせてタイムスタンプが精密に分配されます。
- 表示文字が複数ある場合、内部で等分展開されるため、一文字ずつの評価が可能になります。

### ② 間奏タグ `[開始分:秒 - 終了分:秒]`
長い間奏やコーラス区間を固定します。
- `[01:20.50 - 01:45.00]`
- **分割解析のトリガー:** このタグを入れると、タグの前後が「独立したセグメント」として解析されます。一方のミスが他方に波及しないため、複雑な構成の曲では必須のテクニックです。

## 4. 精度を上げるための戦略（MAEの改善）

アライメントがズレる（ワープする）場合は、`karatag_debug.log` の末尾にある **「品質レポート」** を確認し、以下の手順で対策してください。

### ステップ1：レポートの警告（異常検知）を確認する
1. **「異常高密度」と指摘された場合**: その行の直前でワープが起きています。その付近に **間奏タグ** を挿入して物理的にロックしてください。
2. **「異常低密度」と指摘された場合**: Whisperがノイズを歌と誤認（ハルシネーション）しています。`--ignore` を調整するか、`anchor_score` を上げてください。

### ステップ2：パラメータの微調整 (`settings.ini`)
1. **滑舌やノイズで誤認が多い**: `anchor_score` を `0.70` 程度に下げ、`alignment_mode = ctc` を試してください。
2. **伴奏を歌と誤認する**: `auto_interlude_threshold` を `0.05` 程度に上げてください。
3. **1音が長く伸びる曲**: `max_char_duration` を `2.0` 以上に広げてください。
4. **超早口な曲**: `min_char_duration` を `0.01` まで下げてください。

## 6. 困ったときは
- **解析プロセスの確認**: `karatag_debug.log` には、Whisperが聴き取った全候補（ASR実況）、骨格確定の瞬間、CTC確定の結果が秒単位で記録されています。
- **タグの形式エラー**: 歌詞ファイル内の `[...]` は、必ず `[表示|よみ]` または `[mm:ss - mm:ss]` の形式である必要があります。それ以外の括弧はエラーとなります。
